Tabelas detalhadas

O objetivo desta área é mostrar claramente quais tabelas compõem o runtime operacional real e como elas se conectam ao core sem recriar diretório local de tenant ou usuário.

Configuração e canal

AtendimentoTenantConfigAtual + integração

Configuração operacional local por tenant canônico do core.

  • tenantId string PK
  • retentionDays, maxUploadMb
  • createdAt, updatedAt
{
  "tenantId": "67668cc8-bc63-457d-b6ce-94118c593d6f",
  "retentionDays": 15,
  "maxUploadMb": 500
}

O runtime não replica mais nome, slug, grant base, limite de usuários nem segredo de canal.

WhatsAppInstanceAtual

Instância operacional do canal.

  • tenantId, instanceName, displayName
  • phoneNumber, queueLabel
  • createdByUserId, responsibleUserId
  • isDefault, isActive
{
  "tenantId": "67668cc8-bc63-457d-b6ce-94118c593d6f",
  "instanceName": "demo-instance",
  "displayName": "WhatsApp Demo Core Tenant",
  "isDefault": true
}

As colunas de usuário e tenant guardam ids do core. O ownership local é do canal e do roteamento operacional.

Contato, conversa e mensagem

ContactAtual

Cadastro operacional de contato.

  • tenantId, name, phone
  • avatarUrl, source
ConversationAtual

Conversa consolidada por contato, canal e instância.

  • tenantId, instanceId, instanceScopeKey
  • assignedToId, contactId, channel, externalId
  • status, lastMessageAt
MessageAtual

Mensagem inbound ou outbound da operação.

  • tenantId, conversationId, instanceId
  • senderUserId, direction, messageType
  • content, mídia, metadata e status
{
  "direction": "OUTBOUND",
  "senderUserId": "e0556192-acd1-4774-ac2f-f79410178cf7",
  "content": "Perfeito! Aqui o runtime operacional já está vinculado ao core."
}

A autoria humana usa id de usuário do core; o módulo não replica mais operador local.

Trilha e preferências

AuditEventTécnico-operacional

Trilha operacional ligada a conversa e mensagem.

  • tenantId, actorUserId
  • conversationId, messageId
  • eventType, payloadJson, createdAt
SavedStickerAtual

Biblioteca operacional de figurinhas.

  • tenantId, createdByUserId, name
  • dataUrl, mimeType, sizeBytes
HiddenMessageForUserAtual

Ocultação individual de mensagem para operador do core.

  • tenantId, userId, messageId
  • createdAt

Síntese: o núcleo operacional deste schema faz sentido continuar. O runtime ficou dono apenas do dado operacional; tenant, usuário, auth, grants autoritativos, limites e segredo de canal já estão fora do banco do módulo.
