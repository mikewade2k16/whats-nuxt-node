Schema Finance

Catálogo visual do módulo financeiro extraído do shell

O financeiro saiu do `platform_core` e passou a viver em schema próprio `finance`, mantendo apenas a dependência compartilhada em `platform_core.tenants` para isolamento por cliente. Aqui ficam a configuração estrutural do tenant, as recorrências, a planilha do período, suas linhas e os ajustes detalhados.

Fluxo estrutural

1

Tenant ancora o módulo

`finance_configs` nasce por tenant, mas o owner do tenant continua no shell canônico.

2

Estrutura fixa é montada

Categorias, contas fixas e composição interna definem a base operacional do financeiro.

3

Recorrências se ligam ao tenant

`finance_recurring_entries` referencia tenants do shell para rateios e ajustes recorrentes.

4

Período abre a planilha

Cada fechamento cria uma `finance_sheets` com compatibilidade histórica por `legacy_id`.

5

Linhas e ajustes fecham o saldo

`finance_lines` e `finance_line_adjustments` materializam o lançamento mensal com resumo calculado em SQL.

Fronteira de ownership

Owner do tenant: platform_core Owner do financeiro: finance
  • `platform_core` continua dono de identidade, grants, módulos, limites, auth e auditoria.
  • `finance` fica dono da configuração financeira, da operação mensal e do histórico das planilhas.
  • O schema novo não cria shadow de usuário, tenant ou permissões.
  • A principal fragilidade remanescente é `finance_lines.fixed_account_id`, que segue como soft reference para evitar deadlock em update concorrente.

Tabelas do schema

finance_configsAtual

Configuração financeira única por tenant.

  • `id` uuid PK
  • `tenant_id` uuid FK para `platform_core.tenants` com unique
  • `updated_at`
{
  "tenant": "acme-core"
}
finance_categoriesAtual

Categorias de lançamento do tenant.

  • `id` uuid PK; `config_id` FK
  • `name`, `kind`, `description`, `position`
{
  "name": "Comissões",
  "kind": "saida",
  "position": 2
}
finance_fixed_accountsAtual

Conta fixa configurada por tenant.

  • `id` uuid PK; `config_id` FK
  • `name`, `kind`, `category_id`, `default_amount`, `notes`, `position`
{
  "name": "Comissão mensal loja Jardins",
  "kind": "saida",
  "default_amount": 3500.0
}
finance_fixed_account_membersAtual

Composição ou rateio da conta fixa.

  • `id` uuid PK; `fixed_account_id` FK
  • `name`, `amount`, `position`
{
  "fixed_account": "Comissão mensal loja Jardins",
  "name": "Equipe comercial",
  "amount": 2400.0
}
finance_recurring_entriesAtual

Recorrências entre tenants ou centros de custo.

  • `id` uuid PK; `config_id` FK; `source_tenant_id` FK para `platform_core.tenants`
  • `adjustment_amount`, `notes`
{
  "source_tenant": "root",
  "adjustment_amount": 500.0,
  "notes": "Rateio mensal de ferramenta"
}
finance_sheetsAtual + compatibilidade

Planilha financeira do período.

  • `id` uuid PK; `legacy_id` unique via sequence própria do schema
  • `tenant_id` FK; `title`; `period`; `status`; `notes`
  • `created_at`, `updated_at`
{
  "tenant": "acme-core",
  "legacy_id": 14,
  "title": "Fechamento Abril/2026",
  "period": "2026-04",
  "status": "aberta"
}
finance_linesAtual

Lançamento financeiro da planilha.

  • `id` uuid PK; `sheet_id` FK
  • `kind`, `description`, `category`, `effective`, `effective_date`, `amount`
  • `fixed_account_id` soft ref sem FK
  • `details`, `position`
{
  "sheet": "Fechamento Abril/2026",
  "kind": "saida",
  "description": "Comissão equipe loja Jardins",
  "amount": 4200.0,
  "effective": true
}

Este continua sendo o principal ponto de integridade frágil do módulo financeiro.

finance_line_adjustmentsAtual

Ajuste fino de uma linha financeira.

  • `id` uuid PK; `line_id` FK
  • `amount`, `note`, `date`, `position`
{
  "line": "Comissão equipe loja Jardins",
  "amount": -200.0,
  "note": "Desconto de adiantamento"
}
